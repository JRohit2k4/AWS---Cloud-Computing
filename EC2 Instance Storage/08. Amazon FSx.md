**Amazon FSx**  
Think of FSx = “specialized file systems”  
Use it only when EBS or EFS is NOT a good fit.

---

**What is Amazon FSx?**
- A fully managed service for high-performance, third-party file system
- Used when you need Windows file servers or HPC-level performance
- Exam trigger phrase:
  - *"Managed third-party file system"*
 
---

**FSx for Windows File Server** 

Think: Windows + Corporate file shares

**Key points:**
- Windows-native file system
- Uses SMB protocol
- Uses NTFS
- Works with Windows EC2 instances
- Integrates with Microsoft Active Directory
- Can be accessed:
  - From AWS
  - From on-premises data centers

**Exam keywords:**
  Windows, SMB, NTFS, Active Directory, shared Windows file system

**Use cases:**
- Lift-and-shift Windows apps
- Shared file storage for Windows users
- Corporate file servers

---

**FSx for Lustre**
  
Think: Extreme performance + HPC

**Key points:**
- Designed for High Performance Computing (HPC)
- Extremely high throughput
- Millions of IOPS
- Sub-millisecond latency
- Works with Linux
- Often linked with S3 as backend storage

**Exam keywords:**
- HPC, machine learning, analytics, video processing, financial modeling

**Memory trick:**
Lustre = Linux + Cluster

---

**When to choose what**

| Requirement               | Service             |
| ------------------------- | ------------------- |
| Windows file sharing      | **FSx for Windows** |
| Active Directory          | **FSx for Windows** |
| HPC / ML / analytics      | **FSx for Lustre**  |
| Shared Linux files        | **EFS**             |
| Single-instance disk      | **EBS**             |
| Temporary ultra-fast disk | **Instance Store**  |

---

**Exam Traps to Watch**

- EFS does NOT support Windows
- EBS cannot be shared across instances
- Instance Store is NOT durable
- FSx = managed + specialized
